Launch Handler Examples

This guide shows how to use launch handlers in your project config files. Each entry in your config uses the format: ["Display Name", "path/arguments", "handler"]

Complex Handlers

Python-based handlers with advanced logic. Support additional arguments after the path.

NPM npm
Run npm commands in a terminal. Append the command after the path. Commands: start, dev, build, test, install, or any script name.
Examples:
["Start Server", "~/projects/myapp", "npm"] # npm start ["Dev Server", "~/projects/myapp dev", "npm"] # npm run dev ["Build", "~/projects/myapp build", "npm"] # npm run build ["Run Tests", "~/projects/myapp test", "npm"] # npm test ["Install Deps", "~/projects/myapp install", "npm"] # npm install
Complex
Directory Dev directorydev
Opens a full development environment: file manager + terminal + code editor. Optionally runs npm if a command is specified.
Examples:
["My Project", "~/projects/myapp", "directorydev"] # 3 apps, no npm ["My Project", "~/projects/myapp dev", "directorydev"] # + npm run dev ["My Project", "~/projects/myapp build", "directorydev"] # + npm run build
Complex
SSH Session ssh_session
SSH to a server with optional path and command. If second part starts with /, ~, or . it's treated as a path.
Examples:
["Server", "user@host", "ssh_session"] -> ssh user@host ["Server", "user@host /var/www", "ssh_session"] -> ssh user@host 'cd /var/www && bash' ["Server", "user@host /app ./go.sh", "ssh_session"] -> ssh user@host 'cd /app && ./go.sh' ["Server", "user@host npm start", "ssh_session"] -> ssh user@host 'npm start'
Complex
Terminal Command terminal_cmd
Open terminal at a directory with optional command. Same syntax as ssh_session but for local paths.
Examples:
["Project", "~/project", "terminal_cmd"] -> opens terminal at ~/project ["Run Script", "~/project ./go.sh", "terminal_cmd"] -> cd ~/project && ./go.sh ["Dev Server", "~/project npm run dev", "terminal_cmd"] -> cd ~/project && npm run dev
Complex
Dolphin Tabs dolphin_tabs
Opens multiple folders as tabs in your file manager. Paths are space-separated.
Example:
["My Folders", "~/Documents ~/Projects ~/Pictures", "dolphin_tabs"]
Complex
Rsync Backup rsync_backup
Run rsync with --delete and safety checks. Backs up deleted files to ~/.rsync_backups/. Blocks trailing slashes on source.
Example:
["Backup Docs", "~/Documents ~/backup/Documents", "rsync_backup"]
Complex
Rsync + Identity rsync_backup_id
Rsync with SSH identity file. Format: identity_file source destination.
Example:
["Backup to Server", "~/.ssh/key ~/local user@server:/remote", "rsync_backup_id"]
Complex
Rsync + Identity + Port rsync_backup_id_port
Rsync with SSH identity file and custom port. Format: identity_file port source destination.
Example:
["Backup via Port 2222", "~/.ssh/key 2222 ~/local user@server:/remote", "rsync_backup_id_port"]
Complex

Custom Handlers

User-defined handlers stored in launch_handlers_custom.json. Create and edit via Settings > Launch Handlers tab. Custom handlers can override built-in ones.

Example: My Terminal my_terminal
Custom handler example - open Alacritty at a directory.
JSON Definition:
{ "my_terminal": { "command": ["alacritty", "--working-directory", "{path}"], "description": "Open in Alacritty" } }
Custom
Example: Deploy Script deploy
Custom handler example - run a deploy script in terminal.
JSON Definition:
{ "deploy": { "type": "shell", "command": "cd {path} && ./deploy.sh", "terminal": true, "hold": true, "description": "Run deploy script" } }
Custom

Built-in Handlers

Always available. These use your default desktop applications.

Browser browser
Opens URLs in your default web browser using xdg-open.
Example:
["Documentation", "https://docs.python.org", "browser"]
Built-in
File Manager file_manager
Opens folders in your default file manager (Dolphin, Nautilus, etc.).
Example:
["Downloads", "~/Downloads", "file_manager"]
Built-in
Editor editor
Opens files in your default text editor.
Example:
["Config File", "~/.bashrc", "editor"]
Built-in
Default default
Lets the system decide how to open the file based on its type.
Example:
["Spreadsheet", "~/Documents/budget.xlsx", "default"]
Built-in
Terminal terminal
Opens a terminal window at the specified directory. For running commands, use terminal_cmd instead.
Example:
["Project Terminal", "~/projects/myapp", "terminal"] -> opens terminal at ~/projects/myapp
Built-in

Simple Handlers

Defined in launch_handlers.py. These execute specific commands with the path as an argument.

Firefox firefox
Opens a URL in a new Firefox window.
Example:
["Wikipedia", "https://wikipedia.org", "firefox"]
Simple
Chrome chrome
Opens a URL in Chrome via Flatpak in a new window.
Example:
["Google", "https://google.com", "chrome"]
Simple
Tail Log tail_log
Opens a terminal and tails debug.log in the specified directory.
Example:
["Watch Logs", "~/projects/myapp", "tail_log"]
Simple
Kitty Tail kitty_tail
Same as tail_log but uses kitty terminal with --hold to keep window open.
Example:
["Watch Logs (Kitty)", "~/projects/myapp", "kitty_tail"]
Simple
